## Recommendations

1. **Fix the broken in-soak `hone hunt` step today** — the
   highest-EV single action. `planning/SOAK_FINDINGS.md`
   Finding 2 names three options; (c) "run hone-soak.service
   as `worker`" is the smallest fix and recovers the next ~40 h
   of hunt productivity. Without this the soak's 48 h budget
   produces nothing new and the post-soak DoD (item 4 of
   workspace CLAUDE.md, "≥ 96 hone-soak.timer ticks recorded
   with new findings") fails on substance even if it passes on
   tick count.
2. **File the three soak-surfaced issues as kb findings.**
   `planning/SOAK_FINDINGS.md` Finding 1 (`pkt.proto in [list]`
   interpreter AttributeError), Finding 2 (the hunt-step
   permission breakage), Finding 3 (`bpf_loader.cc` pinned-map
   name drift) should have IDs in `findings/`. The first is
   already fixed; create it AND mark `triage/fixed/` so the kb
   reflects truth. Without this, the harness's own success
   metrics lie.
3. **Implement the `helper-walks-rules-only` strategy** named in
   the prior critique. The pattern file
   `pattern/helper-skips-tier2-function-body` already has the
   prompt written ("Where to Look Next" enumerates six
   helpers). Wire it into `meta/strategy_weights.json` with
   initial weight 0.10, run it twice, expect at least three of
   the six predicted candidates to produce a finding. If it
   produces 0/2, the pattern is wrong, not the surface.
4. **Promote-or-refute the four `status: hypothesis` patterns
   in this round.** For each:
   - `helper-skips-tier2-function-body`: run the strategy from
     #3 above. Result drives promotion to `confirmed` or
     pruning of the pattern's claimed scope.
   - `ipv6-literal-spec-incompleteness` (N=2): force decisions
     on the two cited findings (IPv4-mapped canonical form,
     trailing-colon-vs-`if`-suffix). One spec edit each.
   - `pkt-loader-silent-acceptance` (N=4): write a `.pkt`
     fuzzer (50 cases) that emits unknown builder fields,
     non-canonical IPv6, out-of-range state idx — assert the
     loader rejects every case. Pass = pattern stays open and
     guides future loader edits; fail = file each fail as a
     finding.
   - `v02-rule-incomplete-enumeration` (N=4): the pattern's
     check strategy enumerates 6 mechanical sweeps. Run one of
     them — "for every grammar terminal that overlaps with the
     identifier production, check the spec enumerates whether
     it is reserved" — produces a finite list of unenumerated
     terminals.
5. **Re-prompt `oracle_divergence` and rerun against the soak's
   live `dogfood_v02.fw`.** Prior critique recommendation #3 is
   unaddressed. The strategy ran once at 0 hits; the harness's
   three biggest oracle-divergence bugs (v6-activation, v4_ok
   gate, geoip map pop) all came out of `hunt` instead. The
   prompt is mis-specified. Replace it with: "given the live
   soak's dogfood_v02.fw, build a v4-builder `.pkt` and a
   v6-builder `.pkt` that exercise the same logical path; run
   both through interpreter and through `fd` on deb-02; assert
   the verdict matches." This is exactly what should be running
   under hone-soak ticks in the 48 h window.
6. **Audit `f/src/*.cc` against `pattern/helper-skips-tier2-function-body`.**
   `bpf_loader.cc`'s pinned-map naming
   (`planning/SOAK_FINDINGS.md` Finding 3) is the same shape as
   the four pattern instances on the Python side. Spawn a
   `hone hunt` round whose only target is `f/src/bpf_loader.cc`,
   `f/src/slow_path.cc`, `f/src/engine.cc`, with the prompt:
   "every place this code references a hard-coded map name or
   counter name produced by v0.1 FWL, decide whether the v0.2
   per-rule naming scheme has been honoured." The fix to
   Finding 3 lands in the impl workspace; the kb has to file
   the bug.
7. **Add a `mutate` corpus seed and bounded operator set or
   retire the strategy.** Prior critique recommendation #6 is
   unaddressed. 0/4 over two critiques is dead capacity. Either
   give it the corpus seed (`fwl/tests/corpus/08_ipv6_fields/`,
   `09_geoip/`, `10_tier2_functions/`) plus a four-operator
   restriction (CIDR ±1, sibling-family proto swap, drop
   `pkt.` prefix, `==`↔`in`), run two rounds, expect ≥ 1 hit;
   or remove it from the roster so the tune doesn't keep
   weighting it.
8. **Add a `pattern-replay` strategy** that, every Nth tick,
   walks `patterns/*.md`, picks one whose `instance_count` has
   not grown in 14 days, runs its "Check Strategy" steps, and
   either grows the instance list or marks the pattern stale.
   This converts the patterns from documents into a feedback
   loop. With 11 patterns, one rotation every 11 ticks gives
   each pattern a probe per ~5.5 h of soak time.
9. **Open a finding for the strict-superset implementation
   regress hole.** The spec promise is verified by spec text;
   the impl-level promise is unverified. Prior critique
   recommendation #11 is unaddressed; turning it into a kb
   finding (severity high, layer harness) creates a tracked
   item that survives across critiques rather than re-floating
   each time.
10. **Update the hunt prompt to record numbered hypothesis
    tables in misses, even on no-hit rounds.** Prior critique
    recommendation #9 is unaddressed. Three of the five
    misses are still program-scoped regression-pass write-ups,
    not hypothesis-level refutations. Without this fix the
    harness keeps re-testing already-refuted hypotheses.
